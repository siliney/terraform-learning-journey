Day 10 — Provisioning (install Nginx automatically)
Example setup.sh (Ubuntu) 
#!/usr/bin/env bash
set -euo pipefail

sudo apt-get update -y
sudo apt-get install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx

echo "OK: Nginx installed on $(hostname) at $(date)" | sudo tee /var/www/html/index.html
Provisioners are okay for learning, but in real jobs many teams prefer cloud-init, Ansible, or images. Still: understanding provisioners helps you bridge Terraform + Linux fast.
Day 11 — Linux Hardening (SSH + firewall + users)
Hardening commands (Ubuntu example) 
# Create admin user
sudo adduser adminuser
sudo usermod -aG sudo adminuser

# SSH hardening
sudo sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl reload ssh

# Firewall
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw --force enable
sudo ufw status
Lockout warning: Keep your current SSH session open while testing a second SSH session. If you break SSH, you may need cloud console access to recover.

Recommended Linux scripts (put in /scripts) 
# scripts/backup.sh (example)
#!/usr/bin/env bash
set -euo pipefail

TS="$(date +%Y%m%d-%H%M%S)"
DEST="/opt/backups"
sudo mkdir -p "$DEST"

sudo tar -czf "$DEST/etc-$TS.tar.gz" /etc
sudo tar -czf "$DEST/www-$TS.tar.gz" /var/www

echo "Backup complete: $TS" | sudo tee -a "$DEST/backup.log"
# scripts/monitor.sh (example)
#!/usr/bin/env bash
set -euo pipefail

echo "== Health Check $(date) =="
systemctl is-active nginx || (echo "Nginx is NOT active" && exit 2)
df -h | head -n 5
free -h | head -n 5
echo "OK"
Cron example (daily backup + hourly monitor) 
# Edit crontab
crontab -e

# Daily 2:00 AM backup
0 2 * * * /usr/bin/bash /opt/scripts/backup.sh >> /opt/backups/cron.log 2>&1

# Hourly monitor
0 * * * * /usr/bin/bash /opt/scripts/monitor.sh >> /opt/backups/health.log 2>&1
For a “clean” portfolio, store scripts in your repo but copy them to the VM during provisioning. Keep your private key OUT of the repo.

Verification & Troubleshooting Drills

Verification checklist (every time)

Common issues → quick fixes
  • SSH timeout: check SG/NSG allows your IP, and VM has public IP.
  • Permission denied (publickey): wrong username (Ubuntu often ubuntu), wrong key file, bad permissions on key file.
  • HTTP not loading: open port 80 in SG/NSG, ensure Nginx running.
  • Provisioner fails: SSH connection not ready; add retry/wait or use cloud-init.

Break/Fix drills (do at least 3 times)
